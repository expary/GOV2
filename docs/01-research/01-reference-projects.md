# 01.01 Reference Projects

GOV2 can learn from excellent open-source projects, but references are limited to product thinking, architecture principles, and tradeoff analysis.

## References

- Kubernetes API conventions: https://github.com/kubernetes/community/blob/master/contributors/devel/sig-architecture/api-conventions.md
- Grafana: https://github.com/grafana/grafana
- PocketBase: https://github.com/pocketbase/pocketbase
- Supabase: https://github.com/supabase/supabase
- Casbin RBAC docs: https://casbin.org/docs/rbac
- Ant Design Pro: https://github.com/ant-design/ant-design-pro

## How GOV2 Uses References

Allowed:

- Learn API versioning discipline.
- Learn modular boundaries and extension points.
- Learn RBAC vocabulary and policy concepts.
- Learn admin-console UX patterns.
- Learn single-binary developer experience ideas.

Not allowed:

- Copy source code.
- Copy UI assets or visual identity.
- Copy database schemas.
- Copy generated files.
- Copy proprietary or license-restricted implementations.
- Recreate another project by renaming symbols.

## GOV2 Position

GOV2 is not a clone of any reference project. It is an original MIT-licensed Go + Vue system framework with its own module boundaries, API contracts, database design, and AI programming rules.
